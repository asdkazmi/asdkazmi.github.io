writeData() Method

writeData("Argument A", "Argument B") will be written in HTML and return JSON/(JS Object) data whose value is not a JS object. It will work as a loop. It will initated by a HTML class "write_data"
Argument A is necessary and will be JSON object location.
Argument B is optional and will be option to choose which will be return from JSON Object, Key or Values.

One of the major developement of writeData("main_data['title']") is writeData() method which is very usefull for web development. It will return data from JSON or JS Object data whose value is not a JS object and it will work as a loop.

How it works?

writeData will be initiated by a HTML class write_data which will initiate a loop. The HTML element initiated by write_data class created a clone of itself and then find the writeData() inside it and check how much clone will be created and return. It depends on writeData() called inside this HTML element, that how much JSON key, values or pair will be return. If there are two writeData() are passed inside this HTML element where one returning 5 values while other returning 3 then 5 loops will be created and 5 HTMl element will be return. Inside this HTML element, writeData("Arg A","Arg B") will be passed which return data from JSON or JS Object.

Return Element length depends on writeData() passed in this element and maximum length between these writeData().

In writeData("Arg A","Arg B") method Arg A is necessary and it will be a JSON object location i.e. which JSON object will return or JSON object key. While Arg B is optional and option for to use which keys or values will be return from considered JSON Object location.

Arg A, JSON Object Location

In Arg A we will pass JSON object location or JSON object key whose value is not a JS Object will be return.
When we will choose JSON object key, then value will be return of specific key.
When we will choose JSON object location while not choose any key, then Arg B will be an option to choose what is in specific JSON object, either keys or values or key-value pair and which keys, values or pair.

Arg B, called wd-option is A very Powerful option. It will be only passed when specified JSON location in Arg A is an Array or Object type.

Arg B is called wd-option

Arg A will be used for JSON object location, which can be a string, int, array or JS object. wd-option is best option when we want to return all values/keys/pairs from json object or only our choice value/keys/pairs. In Arg A JSON object location will be passes while in Arg B option will be passed what we want to return from the specific object.

To write Nested writeData(), an HTML element will be initiated by class nested write_data and then writeData() will be passed inside this nested element.
