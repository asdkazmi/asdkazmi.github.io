Introduction to Grids

writeData("main_data['title']") is a responsive framework for web and provide strong responsive grids system which are used for making website responsive which can be use and access on every screen size easily by user. writeData("main_data['title']") grid system works using CSS media quiries.

writeData("main_data['title']") works as a table row and column and provide 12 column grid system for four different types of screen sizes. Column which will be greater than 12 goes into new row. To initiate writeData("main_data['title']") grid, first we will initiate an HTML element with class row and then column will be it's child elements.

As writeData("main_data['title']") classes are defined as using Type-Value-Effect (optional) form. So every of it's grids class will also be defined using above form e.g. (col-3-tb).

Grids classes form will be defined as col-(Number of Columns to be span, Max 12)-(MIn screen size when grid class taken, optional)

Overriding Grid Classes

Overriding of grid classes provide option to manage grid system for different screen sizes i.e. to span different columns for different screen sizes e.g. we can span 3 column of an element for dt screen and 6 column of same element on tb screen size, when screen size is less than dt and greater than mbl.

Priority of Overriding Grid Classes

    Classes will override descendant classes by following priority.

  • ldt
  • dt
  • tb
  • default without using any screen size
